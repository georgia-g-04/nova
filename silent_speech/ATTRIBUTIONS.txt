3. THIRD-PARTY PRETRAINED MODEL ASSET
-------------------------------------------------------------------------------
- File:     models/face_landmarker.task  (~3.7 MB)
- Source:   Google MediaPipe, downloaded once from the public CDN:
            https://storage.googleapis.com/mediapipe-models/face_landmarker/
            face_landmarker/float16/1/face_landmarker.task
- License:  Apache-2.0 (MediaPipe model). No account or login required.
- Use:      On-device inference of 478 facial landmarks per frame, from which
            this project extracts the 40 lip landmarks. Runs fully offline.

- File:     models/vosk-model-small-en-us-0.15/  (~40 MB)
- Source:   Alpha Cephei / Vosk, https://alphacephei.com/vosk/models (public,
            no account).  License: Apache-2.0.
- Use:      Offline speech-to-text with word timestamps (alternate ASR backend).

- Model:    Whisper "base.en" (via faster-whisper / CTranslate2)
- Source:   OpenAI Whisper weights, auto-downloaded once from the Hugging Face
            Hub (public, no account). License: MIT.
- Use:      Offline speech-to-text used to AUTO-LABEL training data when the
            user speaks a word out loud. Primary ASR backend. Runs on CPU.


-------------------------------------------------------------------------------
4. THIRD-PARTY CONSTANTS & DATA
-------------------------------------------------------------------------------
- Lip landmark index lists (OUTER_LIP / INNER_LIP in src/signal_source.py and
  src/lip_landmarks.py) are the canonical MediaPipe FaceMesh 468/478-point
  topology indices for the lip contour. These are published constants that
  identify which landmark numbers correspond to the lips; they are data, not
  copied source code.

- CMU Pronouncing Dictionary (CMUdict): ~126k English word -> ARPAbet phoneme
  pronunciations, used in src/phoneme_lexicon.py for phoneme<->word lookup.
  The CMUdict DATA is under a permissive BSD-2-Clause-style license from
  Carnegie Mellon University and is redistributable (incl. commercially).
  Accessed here via the `cmudict` pip package (v1.1.3), which bundles the data
  and works offline. NOTE: verify the pip wrapper package's own license before
  redistributing; the raw CMUdict data file can be vendored directly instead if
  a permissive-only dependency chain is required.
    Source: http://www.speech.cs.cmu.edu/cgi-bin/cmudict


-------------------------------------------------------------------------------
5. ALGORITHMS IMPLEMENTED FROM STANDARD REFERENCES (original code)
-------------------------------------------------------------------------------
- Kalman filter (src/kalman.py): NumPy implementation of the standard
  linear Kalman filter with a per-channel constant-velocity (white-noise
  acceleration) model.
- Decoder (src/decoder.py): pipeline built on scikit-learn primitives
  (StandardScaler + PCA + SVM) with velocity-augmented, length-normalized
  trajectory features. Original composition.


-------------------------------------------------------------------------------
6. CONCEPTUAL REFERENCES / DESIGN INSPIRATION (read, NOT copied)
-------------------------------------------------------------------------------
These informed the architecture; no code or weights from them are in this repo.
They are recorded for the later EMG phase and for the written report.

- VALLR: Visual ASR Language Model for Lip Reading (ICCV 2025) - phoneme-then-
  LLM decoding architecture. arXiv:2503.21408
- D. Gaddy, "Digital Voicing of Silent Speech" (EMNLP 2020) and "An Improved
  Model for Voicing Silent Speech" (ACL 2021). Code + EMG dataset + weights:
    https://github.com/dgaddy/silent_speech
    https://zenodo.org/records/4064409  (facial EMG dataset)
    https://doi.org/10.5281/zenodo.6747411  (pretrained models)
  Intended as the reference architecture/data for the future EMG modality.
- "Can LLMs Understand Unvoiced Speech?" (2025) - arXiv:2506.00304
- "Silent Speech Interfaces in the Era of LLMs" survey (2026) - arXiv:2603.11877
- Auto-AVSR (mpc001) and AV-HuBERT (Meta) - surveyed as pretrained lip-reading
  options; NOT used in this codebase.
